# 프로젝트 D 개발 일지

---

## 5일차 : 공통 로딩 화면 구현

### 개발 날짜

2026년 7월 18일

---

### 개발 목표

4일차에 구현한 비동기 씬 전환 기능에 공통 로딩 화면을 연결하고, 로딩 진행률 표시와 페이드 효과 및 입력 차단 기능을 구현한다.

---

### 개발 내용

1. LoadingScreen 스크립트를 구현했다.

   - CanvasGroup 기반 화면 표시와 숨김
   - 페이드 인·아웃 처리
   - 진행률 이미지 갱신
   - 진행률 백분율 표시
   - 로딩 중 UI 선택 상태 해제
   - TimeScale 영향을 받지 않는 페이드 처리

2. 공통 LoadingCanvas를 구성했다.

   - 전체 화면 입력 차단 배경
   - LOADING 안내 텍스트
   - 진행률 배경
   - 진행률 채움 이미지
   - 진행률 백분율 텍스트
   - 일반 UI보다 높은 Sort Order 설정

3. SceneLoader를 확장했다.

   - 씬 로딩 전 로딩 화면 페이드 인
   - AsyncOperation 진행률 정규화
   - 자동 씬 활성화 대기
   - 최소 로딩 화면 표시 시간 적용
   - 새로운 씬 활성화 후 페이드 아웃
   - 로딩 실패 시 화면과 상태 복구

4. 입력 차단 기능을 구현했다.

   - isLoading 기반 중복 요청 차단
   - CanvasGroup 기반 포인터 입력 차단
   - EventSystem 선택 상태 초기화

---

### 로딩 진행 구조

씬 전환 요청
→ 로딩 상태 활성화
→ 입력 차단
→ 로딩 화면 페이드 인
→ 비동기 씬 로딩
→ 진행률 표시
→ 최소 표시 시간 확인
→ 새로운 씬 활성화
→ 로딩 화면 페이드 아웃
→ 입력 차단 해제

### 개발 결과

프로젝트 D의 모든 씬 전환에서 공통으로 사용할 로딩 화면 기반을 완성했다.

씬 전환이 시작되면 기존 화면의 입력이 차단되고 로딩 진행률이 표시된다. 새로운 씬이 활성화되면 로딩 화면이 서서히 사라지고 UI 입력이 다시 허용된다.

현재 씬의 크기가 작아 진행률이 빠르게 증가할 수 있으므로 최소 로딩 표시 시간을 적용했다.
